Measuring Campaign Impact with CausalImpact

practical
causal inference

A practical walkthrough of Google’s CausalImpact package — from data to causal claim.
Published

June 11, 2026

The setup

You run an email campaign on week 10. Sales data exists for 20 weeks. Did the campaign cause an increase in sales?

This is exactly the problem CausalImpact (Google, 2014) was built for. It fits a Bayesian structural time series model on the pre-intervention period, then extrapolates what would have happened without the intervention — your counterfactual.

The data

library(CausalImpact)
library(tidyverse)

set.seed(42)

# Simulate 20 weeks of sales
# Campaign starts on week 10
n <- 20
time <- 1:n

# Control series (unaffected by campaign)
control <- 100 + cumsum(rnorm(n, 0, 2))

# Treatment series: follows control, then bumps up after week 10
treatment <- control + rnorm(n, 0, 3)
treatment[10:n] <- treatment[10:n] + 15  # true effect = +15

df <- data.frame(week = time, sales = treatment, control = control)

Running CausalImpact

The package needs: 1. A time series matrix (treatment + optional control series) 2. The pre/post period indices

# Define periods
pre_period  <- c(1, 9)
post_period <- c(10, 20)

# Format for CausalImpact (zoo object)
ts_data <- zoo(
  cbind(sales = df$sales, control = df$control),
  order.by = df$week
)

# Run the model
impact <- CausalImpact(ts_data, pre_period, post_period)

Reading the results

plot(impact)
summary(impact)
summary(impact, "report")

The three panels show:

  • Original — actual vs. predicted (counterfactual) series
  • Pointwise — difference at each time point
  • Cumulative — total accumulated effect

What the numbers mean

Metric Meaning
Average Estimated effect per time point
Cumulative Total effect over the post period
p-value Probability of this effect under no intervention

In our simulation, we injected a true effect of +15 units. CausalImpact should recover something close to that.

Key assumptions

WarningWhen this breaks down
  1. The control series must be unaffected by the treatment.
  2. The relationship between treatment and control must be stable before the intervention.
  3. No other events happen at the same time.

When to use this

CausalImpact is a good fit when:

  • You have high-frequency data (daily/weekly)
  • You have no control group but have related series
  • The intervention has a clear start date

Classic applications: product launches, PR events, policy changes, algorithm updates.

References

  • Brodersen et al. (2015). Inferring causal impact using Bayesian structural time-series models. Annals of Applied Statistics.
  • CausalImpact R package

## The setup

This is exactly the problem `CausalImpact` (Google, 2014) was built for. It fits a Bayesian structural time series model on the pre-intervention period, then extrapolates what *would have happened* without the intervention — your counterfactual.

## When to use this

CausalImpact is a good fit when:

- You have **high-frequency data** (daily/weekly)
- You have **no control group** but have related series
- The intervention has a **clear start date**

Classic applications: product launches, PR events, policy changes, algorithm updates.
